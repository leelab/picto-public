Detailed Description

A StateUpdater that gathers Session events from the Picto Server in real time.

This StateUpdater is used in the Remote Viewer to keep the local slave Experiment in sync with the master Experiment running on the Director. It is used in the RemoteViewer's SlaveExperimentDriver which doesn't care about the StateUpdater implementation. The result is that we can use the same system that we use for Experimental playaback for viewing live Experiments. The only difference is that we don't currently support Analysis of live Experiments. This is partially due to the fact that no future data is available at any given time. It is for this reason that we are returning empty pointers from all of the StateUpdater get?Reader() functions.

bool RemoteStateUpdater::updateState ( )
virtual

Requests all new Session data from the Picto Server and emits the appropriate signals accordingly.

This function keeps track of the latest time for which data has arrived from the Server then requests all data from the Server that occured after that time. The function goes through the returned data and emits the appropriate StateUpdater signals for each type of new data that arrives. This function should be called at least once per frame in order to be sure that the most up to date data is always coming into the Workstation.

Implements Picto::StateUpdater.
