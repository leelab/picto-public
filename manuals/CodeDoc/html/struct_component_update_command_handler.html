Detailed Description

Handles COMPONENTUPDATE commands, which are sent by ComponentInterface objects (Director and Proxy).

ComponentInterface objects (ie. Director, Proxy... not Workstation) send COMPONENTUPDATE commands to the Server at a fixed interval. These provide up to data information about the Component status, keep the network connection from timing out and are used to send directives from the Picto Server to the Component.

QSharedPointer< Picto::ProtocolResponse > ComponentUpdateCommandHandler::processCommand ( QSharedPointer< Picto::ProtocolCommand >  command)
virtual

Parses the input ProtocolCommand to check its syntax, extract the included Component data and use it to update the ConnectionManager and a Session that it is being used in. Returns any pending directives for the calling Component, or a simple OK response otherwise.

Note
This function takes part in making sure the correct registered command responses are sent to Components to let them know which of their commands have been processed and their data saved to disk. See Picto::RegisteredResponseType for more details.
