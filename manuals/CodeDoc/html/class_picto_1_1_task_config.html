Detailed Description

There are many operations and data elements that need to be associated with a task as a whole, accessible by all elements of the task and with access to all elements of the task. Objects of the TaskConfig class handle these types of operations and store this type of data. This is a lower-scoped analog to the DesignConfig class, and much lighter-weight.

Some of the types of operations that require access to/from all task Assets are:

  • Configuring and accessing Task plot objects
  • (FUTURE) Uniquely gating Input vs Output on the NIDaq Digital lines

These and other general task configuration issues are handled by this class. Every Picto Asset holds a pointer to the TaskConfig of the task to which it is attached and elements needing to be accessed through the TaskConfig add themselves to the TaskConfig object (which holds a weak pointer to them to avoid circular shared pointer issues). The TaskConfig of a particular Task can be accessed using its Asset::getTaskConfig() function.
