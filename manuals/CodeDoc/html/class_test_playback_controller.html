Detailed Description

The TestPlaybackController handles the play(), pause(), stop() interface to an Experiment.

This is essentially a convenience class that allows us to just start and stop running Tasks in an Experiment without worrying about details like its interaction with the Engine::PictoEngine.

Member Function Documentation

void TestPlaybackController::play ( )
slot

Starts / Resume playback of the current Task.

When no Task is running, this emits running() then starts running the Task. The function blocks until the Task ends, either on its own, or because stop was pressed. When that happens, the stopped() signal is emitted and the function ends.

If this is called when a Task is already running, it causes the task to resume, and the running() signal is emitted.

Note
Since the Task runs underneath the play() function, when this function is used to resume playback, it is actually being called from below another call to play(). This is confusing, and at some point we need to refactor the Experimental run system so that it is played from within a Qt Event loop.

void TestPlaybackController::stop ( )
slot

Stops the currently running task.

As described in play(), if a Task is running it means that this is getting called somewhere underneath a call to TestPlaybackController::play(). The reults of this function will be that the Experiment::runTask() function in play() will return, and then execution will continue within that function until the stopped() signal is emitted.
