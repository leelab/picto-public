Detailed Description

A plugin allowing the proxy server to interface with a Plexon MAP.

This uses the API provided by Plexon to extract neural data from a running Plexon system. The documentation for this API is relatively easy to come by. All of the functions do what they are supposed to do according to the NeuralDataAcqInterface documentation. This class simply implements those functions for the Plexon interface.

Note
The PlexClient.dll must be in the path for this plugin to be loaded. Currently we place it in the PictoRunPath directory alongside the dll generated for this class.

Causes the interface to throw out all data collected before now.

QList< QSharedPointer< Picto::DataUnit > > PlexonPlugin::dumpData ( )
virtual

Returns a list of DataUnits read from the Neural data device since the last call to this function.

The returned list includes NeuralDataUnit, AlignmentDataUnit, and LFPDataUnitPackage objects. The NeuralDataUnit and AlignmentDataUnit objects represent individual spikes and alignment events, but the LFPDataUnitPackage represents a list of LFP readings for a particular channel. To keep the load on the server low but balance that with fresh data on the workstation, the typical LFPDataUnitPackage should include about a half second's worth of data.
