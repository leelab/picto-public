\subsection{Detailed Description}
The top level object that controls Session playback using the Playback\-State\-Updater and Slave\-Experiment\-Driver. 

The \hyperlink{class_playback_controller}{Playback\-Controller} manages playback of Session files. It runs in its own thread so that long operations like loading don't interfere with the U\-I. Interfacing with the \hyperlink{class_playback_controller}{Playback\-Controller} is through functions like \hyperlink{class_playback_controller_afc291b7029c1eda156aad5187b3c43f6}{play()}, \hyperlink{class_playback_controller_ac0d6f4d474fc5fce75e4d822360dbd08}{pause()}, \hyperlink{class_playback_controller_a15d048af021ac2b4d66e90b5a96c2c3e}{stop()}, \hyperlink{class_playback_controller_a5c751dc9c22b95f6f601380f503f8f9c}{jump\-To\-Time()}, etc. These functions communicate with the parts of the Playback\-C\-Ontroller that run in the \hyperlink{class_playback_controller}{Playback\-Controller} thread through a thread-\/safe interface handled by \hyperlink{class_playback_controller_data}{Playback\-Controller\-Data}.

A number of signals are available for a view into what is happening inside the \hyperlink{class_playback_controller}{Playback\-Controller} as well as a way to gather data about things like \hyperlink{struct_preloaded_session_data}{Preloaded\-Session\-Data}. \begin{DoxyAuthor}{Author}
