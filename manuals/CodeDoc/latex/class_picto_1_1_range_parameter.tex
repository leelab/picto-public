\subsection{Detailed Description}
A \hyperlink{class_picto_1_1_parameter}{Parameter} that holds integer values within a limited range. 

The range is between the Min and Max \hyperlink{class_picto_1_1_property}{Property} values (inclusive). There is also a legacy \hyperlink{class_picto_1_1_property}{Property} called Increment. This \hyperlink{class_picto_1_1_property}{Property} has been set invisible for new versions of \hyperlink{namespace_picto}{Picto} since it is really just confusing and not too useful; however, it has been left around to support older designs. It forces the Value \hyperlink{class_picto_1_1_property}{Property} to be a precise number of Increments from the Min value if used. If set to zero (the default), it is not used.

Currently, the widget representing the \hyperlink{class_picto_1_1_range_parameter}{Range\-Parameter} value in the \hyperlink{class_property_frame}{Property\-Frame} in the \hyperlink{class_test_viewer}{Test\-Viewer} and \hyperlink{class_remote_viewer}{Remote\-Viewer} doesn't appear to be limiting values to be within the Min-\/\-Max range. We would have expected this to work automatically when \hyperlink{class_picto_1_1_range_parameter_a7d3286c6f2fc89604be66fe7314277f1}{fix\-Values()} brings the value back to the allowed range, but there appears to be some issue. This shoul be looked into. \begin{DoxyAuthor}{Author}
