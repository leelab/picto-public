\subsection{Detailed Description}
A transition between two State\-Machine\-Elements

A transition links a specific \hyperlink{class_picto_1_1_result}{Result} value from a specific \hyperlink{class_picto_1_1_state_machine_element}{State\-Machine\-Element} to another \hyperlink{class_picto_1_1_state_machine_element}{State\-Machine\-Element}. Since no two State\-Machine\-Elements on the same level are allowed to have the same name, all of the connected Assets can be referenced by name. We serialize Asset\-Id references and keep pointers to the Assets as well though in order to avoid problems when designers accidentally name two things with the same name (this can also happen briefly if one \hyperlink{class_picto_1_1_state_machine_element}{State\-Machine\-Element}'s name is equal to anothers with additional letters on the end).

Transitions are mainly just containers for Source, Source\-Result and Destination data; however they also include functionality to automatically delete themselves if one of the Assets to which they are connected is deleted. \begin{DoxyAuthor}{Author}
