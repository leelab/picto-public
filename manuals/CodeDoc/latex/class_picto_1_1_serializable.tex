\subsection{Detailed Description}
The top level class of the \hyperlink{namespace_picto}{Picto} Design class hierarchy, and the \hyperlink{namespace_picto}{Picto} network transferrable data hierarchy, providing an interface for serialization/deserialization. 

Everything element of the \hyperlink{namespace_picto}{Picto} design and of network transferrable \hyperlink{class_picto_1_1_data_unit}{Data\-Unit} objects must be savable and loadable, so we put that functionality into this hierarchical root class. An interface to retrieve serialization errors is also provided. \begin{DoxyAuthor}{Author}

Returns the latest deserialization errors that were reported from any deserialization event. 

Since Deserialization of \hyperlink{namespace_picto}{Picto} designs is not something that happens simultaneously in multiple threads, we can just use a static function along with a static list of errors\-\_\- to track serialization errors. The procedure is to call \hyperlink{class_picto_1_1_serializable_acbaf02a158b3440a16bc1a84ebaa4481}{from\-Xml()} on the node that you want to deserialize then if that function returns false, indicating that deserialization falied, to call \hyperlink{class_picto_1_1_serializable_a9b14c771646820d10e082d351cd4cf50}{get\-Errors()} to find out what the errors were and \hyperlink{class_picto_1_1_serializable_a6c5644865540a1e1e8951895c836b5cf}{clear\-Errors()} to clear out the error report for next time. \begin{DoxyNote}{Note}
In the typical scenario, \hyperlink{class_picto_1_1_serializable_acbaf02a158b3440a16bc1a84ebaa4481}{from\-Xml()} would be called with the entire \hyperlink{namespace_picto}{Picto} Design X\-M\-L document, and \hyperlink{class_picto_1_1_serializable_a9b14c771646820d10e082d351cd4cf50}{get\-Errors()} would only be used after the entire design had been loaded to report all of the deserialization errors at once. 
\end{DoxyNote}
