The Direct3\-D Mobile compositing surface is used on Picto\-Box (which is running Windows\-C\-E. Direct3\-D Mobile is used to provide the maximum amount of speed and hardware control. Obviously, this code should only be included if we're building for the embedded system.

This visual target uses the Direct3\-D Mobile interface to present visual stimuli. Direct3\-D Mobile is only supported by Windows C\-E, so this code is only included if the target O\-S is Win\-C\-E.

The other Visual\-Targets allow us to run in windowed mode. However, since this target is going to be exclusively used on Picto\-Box, there is no reason to allow a windowed mode.

This function sets up the port numbers used by \hyperlink{namespace_picto}{Picto} to communicate with other picto applications. The input name should come from the Names object that is set up in \hyperlink{namespace_picto_ab626f6cf7953f6f0c2947f0f03b8e62e}{Initialize\-Lib()}. \begin{DoxyNote}{Note}
When the \hyperlink{namespace_picto}{Picto} \hyperlink{class_server}{Server} runs as a service, it does not have access to the regular \hyperlink{namespace_picto}{Picto} settings so they need to be saved in the System Scope. This is the purpose of the used\-In\-System\-Services parameter. All other \hyperlink{namespace_picto}{Picto} applications should save their values in the user scope (because setting system settings requires administrator previleges).
