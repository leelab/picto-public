Objects of the \hyperlink{class_picto_1_1_design_config}{Design\-Config} class handle these types of operations and store this type of data.

\subsection{Detailed Description}
There are many operations and data elements that need to be associated with a design as a whole, accessable by all elements of the design and with access to all elements of a design. Objects of the \hyperlink{class_picto_1_1_design_config}{Design\-Config} class handle these types of operations and store this type of data. 

Some of the types of operations that require access to/from all design Assets are\-:
\begin{DoxyItemize}
\item Making sure that all Asset\-Ids are unique
\item Implementing \hyperlink{class_picto_1_1_asset}{Asset} lookup based on its Asset\-Id
\item Activating particular Analyses
\item Creating lists of all design \hyperlink{class_picto_1_1_asset}{Asset} information for use in building up a S\-Q\-L session file.
\item Storing objects for accessing data from input sources to be used by Experimental and \hyperlink{class_picto_1_1_analysis}{Analysis} components of the \hyperlink{class_picto_1_1_state}{State} Machine. These and other general design configuration issues are handled by this class. Every \hyperlink{namespace_picto}{Picto} \hyperlink{class_picto_1_1_asset}{Asset} holds a pointer to the \hyperlink{class_picto_1_1_design_config}{Design\-Config} of the design to which it is attached and adds itself to the \hyperlink{class_picto_1_1_design_config}{Design\-Config} object (which holds a weak pointer to it to avoid circular shared pointer issues). The \hyperlink{class_picto_1_1_design_config}{Design\-Config} of a particular \hyperlink{class_picto_1_1_experiment}{Experiment} or overall \hyperlink{class_picto_1_1_picto_data}{Picto\-Data} root object can be accessed using its \hyperlink{class_picto_1_1_asset_a44b0ef3025e5bb5f0712fc176c769e5d}{Asset\-::get\-Design\-Config()} function. \begin{DoxyAuthor}{Author}
